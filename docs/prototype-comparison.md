# Prototype comparison

## A — Character simulation

**Fantasy**: live as a person in a changing world.

**Strengths**:
- immediate narrative attachment
- strong Kenshi / RimWorld / life-sim flavor
- easy to showcase living NPCs

**Risks**:
- content scope can explode quickly
- may dilute the Pokémon angle early

## B — Pokémon / creature embodiment

**Fantasy**: live as a creature, grow, bond, survive, and evolve.

**Strengths**:
- strong identity
- very good fit for memory, instincts, territory, and evolution
- can feel fresh compared to trainer-centric designs

**Risks**:
- requires careful UX for non-human agency
- harder to design menus and long-term objectives cleanly

## C — 4X / settlement / region management

**Fantasy**: shape a settlement, city, region, or civilization.

**Strengths**:
- great fit for systemic simulation and event generation
- scalable to factions, diplomacy, ecology, and history

**Risks**:
- biggest implementation scope
- easiest path to losing the personal “living NPCs” feeling

## Suggested recommendation

Start with a **small persistent simulation slice** that can later branch toward A, B, or C.
